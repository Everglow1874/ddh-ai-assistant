# 数据库设计

## 1. 概述

本项目使用 MySQL 8.0 作为持久化存储，存储元数据、对话记录和作业信息。

- 数据库名：`ddh_assistant`
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_general_ci`

## 2. ER 关系

```
project (1) ──── (N) table_metadata (1) ──── (N) column_metadata
   │
   ├──── (N) chat_session (1) ──── (N) chat_message
   │
   └──── (N) etl_job (1) ──── (N) etl_job_step
                │
                └──── (1) chat_session （来源对话，可选）
```

关系说明：
- 一个项目下有多张源表（table_metadata）
- 一张表下有多个字段（column_metadata）
- 一个项目下有多个对话会话（chat_session）
- 一个对话会话下有多条消息（chat_message）
- 一个项目下有多个 ETL 作业（etl_job）
- 一个作业下有多个步骤（etl_job_step）
- 一个作业可以关联一个来源对话（记录该作业是由哪次对话生成的）

## 3. 表设计详情

### 3.1 项目表（project）

管理不同的数仓项目，每个项目拥有独立的元数据和作业空间。

| 字段名 | 类型 | 是否可空 | 默认值 | 说明 |
|--------|------|---------|--------|------|
| id | BIGINT | NOT NULL | AUTO_INCREMENT | 主键 |
| project_name | VARCHAR(100) | NOT NULL | - | 项目名称 |
| description | VARCHAR(500) | NULL | NULL | 项目描述 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

### 3.2 表元数据（table_metadata）

存储导入的源表信息。

| 字段名 | 类型 | 是否可空 | 默认值 | 说明 |
|--------|------|---------|--------|------|
| id | BIGINT | NOT NULL | AUTO_INCREMENT | 主键 |
| project_id | BIGINT | NOT NULL | - | 所属项目ID |
| table_name | VARCHAR(200) | NOT NULL | - | 表名 |
| table_comment | VARCHAR(500) | NULL | NULL | 表注释/描述 |
| schema_name | VARCHAR(100) | NULL | NULL | 所属Schema |
| source_type | VARCHAR(20) | NOT NULL | 'EXCEL' | 来源类型：EXCEL / DDL |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

索引：
- `idx_project_id` (project_id)
- `uk_project_table` (project_id, schema_name, table_name) UNIQUE

### 3.3 字段元数据（column_metadata）

存储表的字段详细信息。

| 字段名 | 类型 | 是否可空 | 默认值 | 说明 |
|--------|------|---------|--------|------|
| id | BIGINT | NOT NULL | AUTO_INCREMENT | 主键 |
| table_id | BIGINT | NOT NULL | - | 所属表ID |
| column_name | VARCHAR(200) | NOT NULL | - | 字段名 |
| column_type | VARCHAR(100) | NOT NULL | - | 字段类型（如 VARCHAR(50), BIGINT） |
| column_comment | VARCHAR(500) | NULL | NULL | 字段注释 |
| is_primary_key | TINYINT | NOT NULL | 0 | 是否主键：0-否 1-是 |
| is_nullable | TINYINT | NOT NULL | 1 | 是否可空：0-否 1-是 |
| default_value | VARCHAR(200) | NULL | NULL | 默认值 |
| sort_order | INT | NOT NULL | 0 | 排序序号 |

索引：
- `idx_table_id` (table_id)

### 3.4 对话会话（chat_session）

每次 ETL 开发交互对应一个会话，记录对话状态和上下文。

| 字段名 | 类型 | 是否可空 | 默认值 | 说明 |
|--------|------|---------|--------|------|
| id | BIGINT | NOT NULL | AUTO_INCREMENT | 主键 |
| project_id | BIGINT | NOT NULL | - | 所属项目ID |
| session_name | VARCHAR(200) | NULL | NULL | 会话名称（可由首条消息自动生成） |
| current_state | VARCHAR(50) | NOT NULL | 'INIT' | 当前对话状态 |
| context_json | TEXT | NULL | NULL | 对话上下文（JSON格式，见下方说明） |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

索引：
- `idx_project_id` (project_id)

**current_state 枚举值：**

| 状态值 | 含义 |
|--------|------|
| INIT | 初始状态，等待用户输入需求 |
| REQUIREMENT_ANALYSIS | 需求分析阶段 |
| TABLE_RECOMMENDATION | 源表推荐阶段 |
| STEP_DESIGN | 作业步骤拆分阶段 |
| SQL_GENERATION | SQL 生成阶段（可循环多次） |
| SQL_REVIEW | SQL 审查阶段 |
| JOB_ASSEMBLY | 作业组装阶段 |
| DONE | 完成 |

**context_json 结构示例：**

```json
{
  "requirement": {
    "raw_text": "统计每个部门本月销售额排名",
    "parsed": {
      "metrics": ["销售额"],
      "dimensions": ["部门"],
      "filters": ["本月"],
      "calculations": ["求和", "排名"]
    }
  },
  "selected_tables": [
    {
      "table_name": "sales_order",
      "selected_columns": ["order_id", "dept_id", "amount", "order_date"]
    },
    {
      "table_name": "department",
      "selected_columns": ["dept_id", "dept_name"]
    }
  ],
  "join_relations": [
    {
      "left_table": "sales_order",
      "left_column": "dept_id",
      "right_table": "department",
      "right_column": "dept_id",
      "join_type": "LEFT JOIN"
    }
  ],
  "steps": [
    {
      "step_order": 1,
      "step_name": "关联订单与部门",
      "description": "创建临时表，关联 sales_order 和 department",
      "target_table": "tmp_dept_order",
      "status": "SQL_GENERATED"
    }
  ],
  "current_step_index": 0
}
```

### 3.5 对话消息（chat_message）

存储每一条对话消息。

| 字段名 | 类型 | 是否可空 | 默认值 | 说明 |
|--------|------|---------|--------|------|
| id | BIGINT | NOT NULL | AUTO_INCREMENT | 主键 |
| session_id | BIGINT | NOT NULL | - | 所属会话ID |
| role | VARCHAR(20) | NOT NULL | - | 消息角色：user / assistant / system |
| content | TEXT | NOT NULL | - | 消息内容 |
| message_type | VARCHAR(30) | NOT NULL | 'TEXT' | 消息类型 |
| metadata_json | TEXT | NULL | NULL | 附加结构化数据（JSON） |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

索引：
- `idx_session_id` (session_id)

**message_type 枚举值：**

| 类型值 | 含义 | metadata_json 内容 |
|--------|------|-------------------|
| TEXT | 普通文本消息 | 无 |
| REQUIREMENT_PARSED | 需求分析结果 | 解析后的结构化需求 |
| TABLE_RECOMMEND | 源表推荐结果 | 推荐的表和字段列表 |
| STEP_DESIGN | 步骤拆分方案 | 步骤列表 JSON |
| SQL_DDL | 建表 SQL | {step_order, table_name} |
| SQL_DML | 数据处理 SQL | {step_order, table_name} |
| SQL_REVIEW | SQL 审查结果 | 审查建议列表 |
| JOB_SUMMARY | 作业汇总 | 完整作业信息 |

### 3.6 ETL 作业（etl_job）

保存完成的 ETL 作业信息。

| 字段名 | 类型 | 是否可空 | 默认值 | 说明 |
|--------|------|---------|--------|------|
| id | BIGINT | NOT NULL | AUTO_INCREMENT | 主键 |
| project_id | BIGINT | NOT NULL | - | 所属项目ID |
| session_id | BIGINT | NULL | NULL | 来源对话ID |
| job_name | VARCHAR(200) | NOT NULL | - | 作业名称 |
| description | VARCHAR(500) | NULL | NULL | 作业描述 |
| status | VARCHAR(20) | NOT NULL | 'DRAFT' | 状态：DRAFT / CONFIRMED / ARCHIVED |
| version | INT | NOT NULL | 1 | 版本号 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

索引：
- `idx_project_id` (project_id)
- `idx_session_id` (session_id)

### 3.7 作业步骤（etl_job_step）

ETL 作业的每一个执行步骤。

| 字段名 | 类型 | 是否可空 | 默认值 | 说明 |
|--------|------|---------|--------|------|
| id | BIGINT | NOT NULL | AUTO_INCREMENT | 主键 |
| job_id | BIGINT | NOT NULL | - | 所属作业ID |
| step_order | INT | NOT NULL | - | 步骤顺序（从1开始） |
| step_name | VARCHAR(200) | NOT NULL | - | 步骤名称 |
| step_type | VARCHAR(30) | NOT NULL | - | 步骤类型 |
| description | VARCHAR(500) | NULL | NULL | 步骤描述 |
| ddl_sql | TEXT | NULL | NULL | 建表 DDL 语句 |
| dml_sql | TEXT | NULL | NULL | 数据处理 DML 语句 |
| source_tables | VARCHAR(500) | NULL | NULL | 依赖的源表（逗号分隔） |
| target_table | VARCHAR(200) | NULL | NULL | 目标输出表名 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

索引：
- `idx_job_id` (job_id)
- `uk_job_step` (job_id, step_order) UNIQUE

**step_type 枚举值：**

| 类型值 | 含义 |
|--------|------|
| CREATE_TEMP | 创建临时/中间表 |
| TRANSFORM | 数据转换处理 |
| AGGREGATE | 聚合汇总 |
| CREATE_RESULT | 创建最终结果表 |
