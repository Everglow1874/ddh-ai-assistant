## 3. 核心模块设计

### 3.1 统一响应体

```java
public class Result<T> {
    private int code;        // 状态码：200成功，其他失败
    private String message;  // 提示信息
    private T data;          // 响应数据

    public static <T> Result<T> ok(T data);
    public static <T> Result<T> ok();
    public static <T> Result<T> fail(String message);
    public static <T> Result<T> fail(int code, String message);
}
```

### 3.2 元数据管理模块

#### ExcelParserService

负责解析用户上传的 Excel 文件，提取表和字段的元数据信息。

**支持的 Excel 格式：**

格式一：单 Sheet（每行一个字段）
```
| 表名 | 表注释 | 字段名 | 字段类型 | 字段注释 | 是否主键 |
|------|-------|--------|---------|---------|---------|
| order_info | 订单表 | order_id | BIGINT | 订单ID | 是 |
| order_info | 订单表 | user_id  | BIGINT | 用户ID | 否 |
```

格式二：双 Sheet（Sheet1=表信息，Sheet2=字段信息）
```
Sheet1 - 表信息:
| 表名 | 表注释 | Schema |

Sheet2 - 字段信息:
| 表名 | 字段名 | 字段类型 | 字段注释 | 是否主键 | 是否可空 |
```

**核心逻辑：**
1. 读取 Excel 文件
2. 自动识别表头（模糊匹配列名）
3. 按表名分组，构建 TableMetadata + ColumnMetadata 对象
4. 批量入库（事务保证原子性）
5. 返回导入结果（成功/失败数量、错误详情）

#### DdlParserService

解析 GaussDB CREATE TABLE DDL 语句，提取表和字段信息。

**核心逻辑：**
1. 使用 JSqlParser 解析 CREATE TABLE 语句
2. 提取表名、字段名、字段类型
3. 通过正则匹配 COMMENT 注释
4. 构建 TableMetadata + ColumnMetadata 对象

### 3.3 AI 引擎模块

#### LlmService（接口）

```java
public interface LlmService {
    /**
     * 同步调用 LLM
     */
    String chat(String systemPrompt, List<ChatMessage> messages);
    
    /**
     * 流式调用 LLM（SSE）
     */
    Flux<String> chatStream(String systemPrompt, List<ChatMessage> messages);
}
```

#### ClaudeLlmService

通过代理 API（兼容 OpenAI 格式）调用 Claude Opus 模型。

**配置项（application.yml）：**
```yaml
llm:
  api-key: ${LLM_API_KEY}
  base-url: ${LLM_BASE_URL}    # 代理 API 地址
  model: claude-opus-4-20250514       # 模型名称
  max-tokens: 4096
  temperature: 0.7
```

**核心实现：**
- 使用 OkHttp 发送 HTTP 请求
- 请求格式兼容 OpenAI Chat Completions API
- 流式响应使用 SSE 解析，逐 token 返回
- 异常处理：超时重试、API 错误码处理

#### PromptBuilder

负责构建各 Agent 的 Prompt，核心职责：

1. 加载 `resources/prompts/` 下的模板文件
2. 动态注入元数据（表结构、字段信息）
3. 注入对话上下文（已确认的中间结果）
4. 控制 Prompt 长度，避免超出 token 限制

**元数据注入格式：**
```

| amount | DECIMAL(10,2) | 订单金额 |
| dept_id | INT | 部门ID |

| 字段名 | 类型 | 说明 |
```

### 3.4 Agent 设计

所有 Agent 继承自 `BaseAgent`，统一接口：

```java
public abstract class BaseAgent {
    
    protected final LlmService llmService;
    protected final PromptBuilder promptBuilder;
    
    /**
     * 执行 Agent 任务
     * @param context 当前对话上下文
     * @param userMessage 用户输入
     * @return Agent 响应结果
     */
    public abstract AgentResult execute(ChatContext context, String userMessage);
    
    /**
     * 流式执行 Agent 任务
     */
    public abstract Flux<String> executeStream(ChatContext context, String userMessage);
}
```

### 3.5 对话状态机

```java
public class ConversationStateMachine {
    
    // 状态转移表
    private static final Map<ConversationState, Map<Action, ConversationState>> TRANSITIONS = Map.of(
        INIT, Map.of(
            SEND_MESSAGE, REQUIREMENT_ANALYSIS
        ),
        REQUIREMENT_ANALYSIS, Map.of(
            CONFIRM, TABLE_RECOMMENDATION,
            REVISE, REQUIREMENT_ANALYSIS
        ),
        TABLE_RECOMMENDATION, Map.of(
            CONFIRM, STEP_DESIGN,
            REVISE, TABLE_RECOMMENDATION,
            GO_BACK, REQUIREMENT_ANALYSIS
        ),
        STEP_DESIGN, Map.of(
            CONFIRM, SQL_GENERATION,
            REVISE, STEP_DESIGN,
            GO_BACK, TABLE_RECOMMENDATION
        ),
        SQL_GENERATION, Map.of(
            CONFIRM_STEP, SQL_GENERATION,     // 确认当前步骤，继续下一步
            CONFIRM_ALL, SQL_REVIEW,          // 所有步骤确认完毕
            REVISE, SQL_GENERATION,
            GO_BACK, STEP_DESIGN
        ),
        SQL_REVIEW, Map.of(
            CONFIRM, JOB_ASSEMBLY,
            REVISE, SQL_GENERATION,
            GO_BACK, SQL_GENERATION
        ),
        JOB_ASSEMBLY, Map.of(
            CONFIRM, DONE
        )
    );
    
    /**
     * 执行状态转移
     */
    public ConversationState transition(ConversationState current, Action action);
    
    /**
     * 获取当前状态可执行的操作
     */
    public List<Action> getAvailableActions(ConversationState state);
}
```

### 3.7 SSE 流式输出

对话接口使用 Server-Sent Events（SSE）实现流式输出，让用户能实时看到 AI 的生成过程。

```java
@GetMapping(value = "/sessions/{id}/send", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public SseEmitter sendMessage(@PathVariable Long id, @RequestParam String message) {
    SseEmitter emitter = new SseEmitter(300_000L); // 5分钟超时
    
    // 异步执行 AI 调用
    executorService.execute(() -> {
        try {
            // 1. 确定当前状态对应的 Agent
            // 2. 流式调用 LLM
            // 3. 逐 token 发送 SSE 事件
            // 4. 完成后发送结构化数据事件
            // 5. 更新对话状态和上下文
        } catch (Exception e) {
            emitter.completeWithError(e);
        }
    });
    
    return emitter;
}
```

**SSE 事件类型：**

| 事件名 | 说明 | data 格式 |
|--------|------|----------|
| message | AI 文本流（逐 token） | 文本片段 |
| structured | 结构化数据（表推荐、步骤方案等） | JSON |
| state | 状态变更通知 | {"state": "TABLE_RECOMMENDATION", "actions": [...]} |
| error | 错误信息 | {"code": 500, "message": "..."} |
| done | 流结束 | 无 |
