# DDH-Assistant 项目概述

## 1. 项目背景

在数据仓库开发过程中，ETL 作业开发是一项核心且高频的工作。开发人员通常需要：

1. 根据业务需求，从大量源表中找到相关表
2. 分析表之间的关联关系和所需字段
3. 判断是否需要创建临时表存储中间数据
4. 编写多步 SQL 脚本（分组、求和、排序、递归求和等）
5. 最终组装成一套有序的作业流，产出结果表

这个过程重复性高、容易出错，且对开发人员的 SQL 能力和业务理解能力要求较高。

**DDH-Assistant（数仓开发助手）** 旨在通过 AI 辅助，将上述流程自动化、标准化，大幅提升 ETL 作业开发效率。

## 2. 产品定位

一个基于 AI 多轮对话的数仓 SQL 作业流开发平台。用户导入源表元数据后，通过与 AI 分步交互，自动完成：

```
需求分析 → 源表推荐 → 中间表设计 → SQL 编写 → 作业流编排
```

最终产出一套可在 GaussDB 计算集群上执行的完整 SQL 作业脚本。

## 3. 核心功能

| 功能模块 | 说明 |
|---------|------|
| 元数据管理 | 通过 Excel/DDL 文件导入源表元数据（表名、字段、类型、注释），支持增删改查 |
| AI 对话交互 | 多轮分步引导式对话，逐步完成从需求到 SQL 的全过程 |
| SQL 生成 | AI 自动生成 GaussDB SQL（DDL 建表 + DML 数据处理） |
| SQL 审查 | AI 对生成的 SQL 进行语法、性能、规范审查 |
| 作业编排 | 将多步 SQL 组装为有序的作业流，展示步骤依赖关系 |
| 作业管理 | 作业的保存、编辑、版本管理、导出（SQL 文件包） |

## 6. 核心交互流程

用户与 AI 助手的完整交互流程如下：

```
┌─────────────────────────────────────────────────┐
│  用户导入源表元数据（Excel / DDL 文件）           │
└─────────────────────┬───────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  Step 1: 需求输入                                │
│  用户用自然语言描述统计需求                        │
│  例: "统计每个部门本月的销售额，并按金额排名"       │
└─────────────────────┬───────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  Step 2: AI 需求分析                              │
│  AI 解析需求，提取：                               │
│  - 目标指标（销售额）                              │
│  - 维度（部门、月份）                              │
│  - 计算方式（求和、排名）                          │
│  - 过滤条件（本月）                               │
│  用户确认或补充                                    │
└─────────────────────┬───────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  Step 3: AI 源表推荐                              │
│  AI 基于元数据，推荐相关源表和字段                   │
│  - 推荐表: sales_order, department                │
│  - 关键字段和关联关系                              │
│  用户确认或修改                                    │
└─────────────────────┬───────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  Step 4: AI 作业步骤拆分                          │
│  AI 设计实现方案，拆分为多个步骤：                   │
│  - Step 1: 创建临时表，关联订单和部门               │
│  - Step 2: 按部门汇总销售额                        │
│  - Step 3: 排名并写入结果表                        │
│  用户确认或调整                                    │
└─────────────────────┬───────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  Step 5: AI 逐步生成 SQL                          │
│  为每个步骤生成 DDL + DML：                        │
│  - CREATE TABLE tmp_xxx ...                      │
│  - INSERT INTO tmp_xxx SELECT ...                │
│  用户逐步确认或修改                                │
└─────────────────────┬───────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  Step 6: AI SQL 审查                              │
│  AI 对所有 SQL 进行审查：                          │
│  - 语法正确性                                     │
│  - 性能优化建议                                   │
│  - 规范性检查                                     │
└─────────────────────┬───────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  Step 7: 作业组装与保存                            │
│  将所有步骤组装为完整作业流                         │
│  可导出为 SQL 文件包，由用户手动在 GaussDB 执行      │
└─────────────────────────────────────────────────┘
```
