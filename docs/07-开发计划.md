# 开发计划

## 1. 环境准备

### 1.1 Java 8 安装

- **版本**：JDK 1.8（推荐 Oracle JDK 8u381 或 Adoptium/Temurin JDK 8）
- **安装目录**：`C:/app/java8`
- **环境变量**：
  - `JAVA_HOME` = `C:/app/java8`
  - `PATH` 追加 `%JAVA_HOME%/bin`
- **验证**：`java -version` 输出 `1.8.x`

### 1.2 MySQL 8.0 安装

- **版本**：MySQL 8.0.x
- **安装目录**：`C:/app/mysql8`
- **配置**：
  - 端口：3306
  - root 密码：自行设置
  - 字符集：utf8mb4
- **初始化操作**：
  1. 创建数据库：`CREATE DATABASE ddh_assistant DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;`
  2. 创建用户：`CREATE USER 'ddh'@'localhost' IDENTIFIED BY 'ddh123456';`
  3. 授权：`GRANT ALL PRIVILEGES ON ddh_assistant.* TO 'ddh'@'localhost';`
  4. 执行建表脚本：`source database-init.sql`
- **验证**：`mysql -u ddh -p -e "SHOW TABLES FROM ddh_assistant;"`

### 1.3 Maven 配置

- 确认 Maven 已安装：`mvn -version`
- 如需配置阿里云镜像仓库，修改 `settings.xml`：
  ```xml
  <mirror>
    <id>aliyunmaven</id>
    <mirrorOf>*</mirrorOf>
    <name>阿里云公共仓库</name>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
  ```

### 1.4 Node.js 环境

- 确认 Node.js 已安装：`node -v`（建议 18.x+）
- 确认 npm 已安装：`npm -v`
- 可选：配置 npm 淘宝镜像
  ```bash
  npm config set registry https://registry.npmmirror.com
  ```

### 1.5 LLM API 配置

- **代理 API Base URL**：待用户提供
- **API Key**：待用户提供
- **模型名称**：`claude-opus-4-20250514`（或代理指定的模型 ID）
- 配置方式：写入 `application-dev.yml` 或通过环境变量注入

## 4. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| AI 生成的 SQL 质量不稳定 | 用户需要大量手动修改 | 持续优化 Prompt；增加 SQL 语法校验；提供便捷的编辑功能 |
| GaussDB 特有语法 AI 不熟悉 | 生成的 SQL 无法在 GaussDB 执行 | 在 Prompt 中明确 GaussDB 语法规范；收集 GaussDB 语法文档作为参考 |
| Token 超限（表数量过多） | AI 无法处理全部元数据 | 实现元数据分批注入策略；让用户先手动筛选相关表 |
| 代理 API 不稳定 | 对话中断或超时 | 实现重试机制；SSE 断线重连；友好的错误提示 |
| 前后端开发进度不同步 | 联调延迟 | 先定义 API 接口文档；后端先用 Mock 数据；前端使用 Mock Server |
