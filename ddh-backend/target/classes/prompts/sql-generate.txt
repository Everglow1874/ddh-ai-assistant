## 重要背景

这是一个 **GaussDB 数据仓库 ETL 作业**，会被调度系统定期（通常每天）自动执行。作业必须包含完整的 **E-T-L 三个阶段**：

### ETL 三阶段规范

1. **Extract（抽取）**：从 ODS/源表中抽取数据到临时表
   - 使用 `CREATE TEMP TABLE ... ON COMMIT PRESERVE ROWS` 创建临时抽取表
   - 支持增量抽取（按日期过滤）或全量抽取
   - 临时表命名：`temp_extracted_xxx`

2. **Transform（转换）**：对抽取的数据进行清洗、转换、标准化
   - 使用 `CREATE TEMP TABLE ... ON COMMIT PRESERVE ROWS` 创建临时转换表
   - 包括：字段清洗（TRIM/COALESCE）、状态码转换（CASE WHEN）、日期处理、业务指标计算、数据质量标记
   - 添加 ETL 元数据字段：`etl_job_name`, `etl_time`
   - 临时表命名：`temp_transformed_xxx`

3. **Load（加载）**：将转换后的数据加载到目标表
   - 使用 `CREATE TABLE IF NOT EXISTS` 创建目标表（含主键约束）
   - 先 DELETE 目标表中对应时间段的旧数据（保证幂等可重跑）
   - 再 INSERT 转换后的数据
   - 最后 DROP 临时表释放资源
   - 目标表命名：`dwd_xxx` 或 `dws_xxx`（明细层/汇总层）

## GaussDB SQL 规范

1. 数据类型：INTEGER, BIGINT, NUMERIC(p,s), VARCHAR(n), DATE, TIMESTAMP, BOOLEAN
2. 常用函数：
   - 日期：CURRENT_DATE, CURRENT_TIMESTAMP, DATE_TRUNC(), EXTRACT(), TO_CHAR()
   - 清洗：TRIM(), COALESCE(), NULLIF(), CAST()
   - 聚合：SUM(), COUNT(), AVG(), MAX(), MIN()
   - 窗口：ROW_NUMBER() OVER(), RANK() OVER()
   - 加密/哈希：MD5()
3. 编码规范：
   - 关键字大写：SELECT, FROM, WHERE, GROUP BY, LEFT JOIN
   - 表别名用小写
   - 每个字段独占一行
   - 使用 `--` 单行注释
   - 缩进 4 个空格
