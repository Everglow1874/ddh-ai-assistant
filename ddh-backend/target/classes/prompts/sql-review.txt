## ETL 规范检查要点

### 1. ETL 三阶段完整性
- 是否包含完整的 Extract → Transform → Load 三个阶段
- 是否正确使用临时表（CREATE TEMP TABLE ... ON COMMIT PRESERVE ROWS）
- 是否在作业结束时清理临时表（DROP TABLE IF EXISTS）
- 目标表是否使用 CREATE TABLE IF NOT EXISTS

### 2. 抽取阶段（Extract）
- 增量条件是否正确（日期范围、时间戳过滤）
- 是否有必要的数据过滤（排除无效数据）
- 字段列表是否完整

### 3. 转换阶段（Transform）
- 数据清洗是否充分（TRIM/COALESCE 处理 NULL 和空格）
- 状态码/枚举值是否正确映射（CASE WHEN）
- 日期字段处理是否合理
- 计算字段逻辑是否正确（注意除零、类型转换）
- 是否添加了 ETL 元数据字段（etl_job_name, etl_time）
- 是否有数据质量标记（is_valid）

### 4. 加载阶段（Load）
- 是否先 DELETE 再 INSERT（保证幂等可重跑）
- DELETE 的 WHERE 条件是否与抽取范围一致
- INSERT 的字段是否与目标表匹配

### 5. SQL 语法和性能
- SQL 语法是否符合 GaussDB 规范
- JOIN 条件是否完整
- 是否有不必要的全表扫描
- 数据类型是否一致

### 6. 编码规范
- 命名是否清晰（temp_extracted_xxx / temp_transformed_xxx / dwd_xxx）
- 注释是否充分
- 格式是否整齐
